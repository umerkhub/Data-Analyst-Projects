# ✈️ Flight Operations Performance Dashboard (Power BI + SQL)

## 📌 Project Overview
This project analyzes airline operations data to uncover patterns in flight volume, delays, and airport efficiency. The goal is to simulate a real-world business intelligence workflow using SQL for data preparation and Power BI for visualization.

---

## 🛠️ Tech Stack
- SQL (MySQL) – Data cleaning & feature engineering  
- Power BI – Dashboard & data visualization  
- DAX – KPI calculations  

---

## 📂 Dataset
- Flight operations data (2024)
- Includes: origin, departure time, delays, taxi time, cancellations

---

## ⚙️ Data Preparation (SQL)
Created a cleaned dataset with key engineered features:
- `total_delay` → combined delay metric  
- `departure_hour` → extracted from time  

---

## 📊 Dashboard Features

### 🔹 KPIs
- Total Flights
- On-Time %
- Average Delay
- Cancellation Rate

### 🔹 Visual Analysis
- Flights Over Time (trend analysis)
- Delay Trend (daily patterns)
- Flights by Airport (hub concentration)
- Flights by Hour (operational patterns)
- Taxi Time Analysis (airport efficiency)

---

## 🔍 Key Insights

- Flight traffic is concentrated in major hubs (ATL, DFW), indicating a hub-driven network where disruptions can impact overall operations.  
- Flights peak between 6 AM–8 PM with early morning spikes, showing airlines optimize for aircraft utilization.  
- Minimal night operations reflect demand and regulatory constraints.  
- Variations in taxi time suggest differences in airport congestion and ground efficiency.
